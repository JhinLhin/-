#### Mistake 失败中学到的教训

- **Scenarios:** There have been plenty of times when I didn’t meet my own expectations. For example, I did not do well on an exam, saying something I could have phrased better, or not handling a situation with a customer in the best way.

It’s totally okay to feel bad or even a little depressed when things don't go well. That’s what makes us different from machines. ### **Scrum:**

- Scrum is an Agile framework that helps manage software projects. It breaks project into smaller part called sprints, which usually last for a few weeks. It involves three main roles: You have the **Product Owner**, who prioritizes tasks based on the needs; the **Scrum Master**, who makes sure everyone follows the process smoothly; and the **Development Team**, who builds the product. 

- 下面的如果问了才说不然显得假：Work is organized in a **Product Backlog** (all the features and tasks) and a **Sprint Backlog** (what we plan to do in the current sprint). Key meetings include **Sprint Planning**, **Daily Stand-Ups** to track progress, **Sprint Reviews** to get feedback, and **Retrospectives** to improve for next time.

### **Git:**

- Git has **four stages**:
  1. **Working Directory**: Where you modify files locally.
  2. **Staging Area**: Use `git add` to stage changes for the next commit.
  3. **Local Repository**: Commit staged changes using `git commit`, creating a version in your local repo.
  4. **Remote Repository**: Push local commits to a shared remote repo (e.g., GitHub) using `git push` for collaboration and backup.

1. **Repositories**:

- **Repository** stores project files and history, either **local** or **remote** (e.g., GitHub).
- **Clone**: `git clone` copies a remote repo to your local machine.

2. **Staging and Commits**:

- **Staging Area**: `git add <filename>` adds files to prepare for commit.
- **Commit**: `git commit -m "message"` creates a version snapshot.

3. **Branches**:

- **Branch**: `git branch <branch-name>` creates a branch for separate work. Switch with `git checkout`.
- **Merge**: `git merge <branch-name>` integrates changes from one branch to another.

4. **Pull and Push**:

- **Push**: `git push origin <branch-name>` shares commits to a remote repo.
- **Pull**: `git pull` fetches and merges changes from the remote repo.

5. **Status and Logs**:

- **Status**: `git status` shows the state of your working directory.
- **Log**: `git log` displays commit history.

6. **Fetch and Rebase**:

- **Fetch**: `git fetch` downloads changes without applying them.
- **Rebase**: `git rebase <branch-name>` repositions commits to keep branches up to date.

7. **Undoing Changes**:

- **Reset**: `git reset` undoes commits.
- **Revert**: `git revert <commit>` undoes a specific commit while preserving history.
- **Checkout**: `git checkout -- <filename>` discards changes in a file.

**Git Workflow**:

- Create **feature branches** for individual tasks.
- Use **pull requests** for code reviews before merging.

### **REST(RESTFUL) API(representational state transfer):**

- RESTful is a software architectural style based on HTTP ,and it is a standard design principle.

- It teaches you => how to design access paths, set request parameter types, and choose request methods.

- Each resource is identified by URI, and we access them through URL:**URI 就像人的名字，用来识别谁是谁，而 URL 就像地址，告诉你如何找到这个人。URL is a subset of URI. **Every URL is a URI because it identifies and locates a resource. Not every URI is a URL.

- 使用标准的 HTTP 方法:

  - `GET`: 用于获取资源。可以通过浏览器地址栏直接输入 URL 来发起。问号后面加参数
  - `POST`: 用于创建新资源。需要通过表单提交、JavaScript、或工具（**如 Postman**）来发起。
  - `PUT`: 用于更新资源。需要通过表单提交、JavaScript、或工具（**如 Postman**）来发起。
  - `DELETE`: 用于删除资源。需要通过表单提交、JavaScript、或工具（**如 Postman**）来发起。

- **资源的表现形式是 JSON 或 XML**: 数据通常以 JSON 或 XML 格式传输。

- 传统风格的 URL 通常带有动词，如 `/CRUD/saveEmp`，而 RESTful 风格的 URL 使用名词表示资源，如 `/CRUD/emp`。

  | 操作 | 传统风格                  | RESTful 风格                          |
  | ---- | ------------------------- | ------------------------------------- |
  | 保存 | `/CRUD/saveEmp`           | URL: `/CRUD/emp` 请求方式: `POST`     |
  | 删除 | `/CRUD/removeEmp?empId=2` | URL: `/CRUD/emp/2` 请求方式: `DELETE` |
  | 更新 | `/CRUD/updateEmp`         | URL: `/CRUD/emp` 请求方式: `PUT`      |
  | 查询 | `/CRUD/editEmp?empId=2`   | URL: `/CRUD/emp/2` 请求方式: `GET`    |
